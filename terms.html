Условия использования

Discord-бота Argus_Bot

Последнее обновление: 28.07.2025
Внимание: Используя Argus_Bot, вы соглашаетесь с этими условиями. Если вы не согласны, пожалуйста, не добавляйте бота на свой сервер.

1. Основные положения

1.1. Определения

  • Бот — программное обеспечение Argus_Bot для платформы Discord
  • Разработчик — создатель и владелец бота (direzon_95)
  • Пользователь — любой пользователь Discord, взаимодействующий с ботом
  • Администратор — пользователь с правами управления сервером Discord

1.2. Общие условия

  • Бот предоставляется на условиях максимально возможной стабильности работы, но мы не можем гарантировать бесперебойную работу в 100% случаев из-за факторов вне нашего контроля (например, перебои в работе Discord или интернет-соединения)
  • Мы прилагаем все усилия для поддержания стабильной работы бота и оперативно устраняем возникающие проблемы
  • Автоматическое создание лог-каналов для модерации (вход/выход пользователей, история сообщений, ролей и голосовые события) с возможностью гибкой настройки через команды !logsettings и !memberlogs
  • Разработчик оставляет право изменять функционал без предупреждения
  • Ваши отзывы и предложения помогают нам улучшать бота
  • Функция поздравлений с днями рождения требует хранения дат рождения пользователей в формате ДД.ММ.ГГГГ и выбранного часового пояса
  • Администраторы сервера могут настроить канал для поздравлений с помощью команды !setup_birthdays
  • Использование бота регулируется Условиями Discord
  • Функции интеграции со Steam предоставляют уведомления о:
    • Новых релизах игр
    • Скидках в Steam Store
    • Активности друзей в Steam
  • Бот не является официальным клиентом Steam и не связан с Valve Corporation

    8.2. Рекомендации

    • Выделенная роль для бота выше пользовательских
    • Ограничение прав по принципу минимальной необходимости
    • Регулярное обновление бота до последней версии

    8.3. Логирование и мониторинг

    • Бот предоставляет инструменты для создания и настройки специализированных лог-каналов
    • Администраторы могут гибко настраивать логирование событий с помощью команд:
      • !memberlogs setup - создание каналов для логирования входа/выхода
      • !memberlogs settings join on/off - включение/выключение логирования входа участников серверов
      • !memberlogs settings leave on/off - включение/выключение логирования выходов участников серверов
      • !logsettings messages on/off - включение/выключение логирования сообщений
      • !logsettings roles on/off - управление логированием изменений ролей
      • !logsettings voice on/off - настройка логирования голосовых каналов
    • По умолчанию все виды логирования отключены и требуют явного включения
    • Для работы функций требуется право Manage Channels
