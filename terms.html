2.2. Ответственность администраторов

  • Администраторы серверов несут ответственность за действия пользователей с ботом
  • Рекомендуется ограничивать доступ к командам модерации
  • При использовании команд модерации (!mute, !tempban и др.) необходимо указывать корректные сроки и обоснованные причины
  • Запрещено использовать систему временных наказаний для злоупотребления властью
  • Запрещено использовать бота для обхода блокировок Discord
Пример: Если пользователь использует бота для спама, администратор сервера должен ограничить ему доступ к боту.

2.3. Безопасность файлов

  • Все файлы, загружаемые через бота, автоматически сканируются антивирусом Dr.Web
  • Запрещена преднамеренная загрузка вредоносных файлов
  • Подозрительные файлы могут быть заблокированы автоматически
  • Система сканирует как пользовательские файлы, так и системные компоненты

4.3. Система временных наказаний

⏱️

Принцип работы

  • Ролевая система Для мута создаются специальные роли
  • Автоматическое снятие Наказания снимаются по истечении срока
🔧

Команды модерации

  • !mute Выдает текстовый мут
    !mute @user 1d10h "Причина"
  • !voicemute Мут в голосовых каналах
    !voicemute @user 2h "Шум в канале"
  • !tempban Временный бан участника
    !tempban @user 3d "Грубое нарушение"
🔄

Снятие наказаний

  • !unmute Снимает текстовый мут
    !unmute @user
  • !voiceunmute Снимает голосовой мут
    !voiceunmute @user
  • !untempban Снимает временный бан
    !untempban @user
Важно: Для работы системы временных наказаний боту требуются права Manage Roles и Moderate Members.

8.3. Логирование и мониторинг

  • Бот предоставляет инструменты для создания и настройки специализированных лог-каналов
  • Администраторы могут гибко настраивать логирование событий с помощью команд:
    • !memberlogs setup - создание каналов для логирования входа/выхода
    • !memberlogs settings join on/off - включение/выключение логирования входа участников серверов
    • !memberlogs settings leave on/off - включение/выключение логирования выходов участников серверов
    • !logsettings messages on/off - включение/выключение логирования сообщений
    • !logsettings roles on/off - управление логированием изменений ролей
    • !logsettings voice on/off - настройка логирования голосовых каналов
  • По умолчанию все виды логирования отключены и требуют явного включения
  • Для работы функций требуется право Manage Channels
